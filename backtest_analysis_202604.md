## 发现的问题

### 一、OBV 因子完全未使用

整份代码没有任何 OBV（On-Balance Volume）的计算或引用。当前评分体系**仅有价格和动量指标，完全没有成交量因子**。

**影响**：
- 无法区分"价涨量增"的真实突破和"价涨量缩"的虚假拉升
- 尾盘拉升信号（+15分）缺少成交量确认，容易出现假信号

### 二、尾盘拉升阈值过低

```python
tail_up = cur_close > cur_open * 1.002  # 仅需 0.2%
```

只需收盘价比开盘价高 0.2% 即可获得 **+15 分**，这是评分体系中第三高的单项加分。实际操作中，这个条件几乎全天大部分股票都满足，缺乏区分度。

### 三、涨幅范围过宽

1%~5% 的涨幅统一给 +20 分。但 14:30 时涨幅已达 4%~5% 的股票，次日回调概率较大；涨幅 0.5%~1.5% 的温和上涨反而更有持续性。

从交易记录看，最大盈利单 **002025（+15.3%）** 买入时涨幅仅 +1.06%。说明大涨的票不一定好，当前评分体系没有区分"温和上涨"和"追高"。

### 四、止盈 +2% 过度保守

这是当前策略**最大的收益限制**：

- 23 笔盈利交易平均 +486（约 +2.3%），几乎刚好在 +2% 被截断
- 少数突破止盈的大赚（002025 +15.3%、603501 +10.8%、300014 +6.8%）均是通过跳空高开直接越过 2%
- **这意味着 2% 止盈只限制了缓慢上涨的利润，而对跳空高开没影响**

典型例子：002422 在 4/15 买入，4/17 以 +2.0% 止盈（+360元），但该股当天全天继续上涨。如果止盈放宽，这部分利润本可保留。

同时 +2% 止损不变，说明止盈是在**不减少亏损的情况下截断了收益**。

### 五、10:00 强制平仓浪费了 T+2

T+2 的持有期本应给持仓两天时间发展，但卖出日只在 9:30~10:00（30 分钟窗口）检查 TP/SL，然后强制卖出。

典型例子：
```
4/10 10:00平仓 002152 +15   — 盈利几乎为零
4/10 10:00平仓 002230 +45   — 微量盈利
4/10 10:00平仓 600036 -148   — 亏损
```

如果卖出检查延长至 14:00，这些头寸有更多机会触达止盈。

### 六、Kronos 预测周期与持仓周期不匹配

```python
pred_len = 5  # Kronos 预测未来 5 天
```

Kronos 预测的是未来 5 个交易日的平均价格，但策略实际持仓仅 2 天。两者时间窗口不一致，可能导致预测信号噪音过大。

## 改进建议（按优先级排序）

### 1. 放宽止盈（最低过拟合风险）

| 当前 | 建议 |
|---|---|
| 止盈 +2% | 止盈 +5% 或取消止盈 |
| 止损 -2% | 保持不变 |

**理由**：止损不变，风险敞口不变，只是让利润奔跑。这不是参数调优，而是消除一个不必要的限制条件。从数据看，亏损全部由止损和 10:00 平仓处理，2% 止盈对控制风险没有贡献。

### 2. 引入 OBV 成交量确认

加入 OBV 作为成交量因子，用于验证尾盘拉升的真实性：

- OBV 上升 + 尾盘拉升 → 确认买入信号 +15 分
- OBV 下降 + 尾盘拉升 → 量价背离，不加分

OBV 是经典量价分析指标，对所有股票普遍有效，不属于过拟合。同时可以补偿当前评分体系完全缺失的成交量维度。

### 3. 延长卖出检查时间

将 10:00 强制平仓改为 14:00 强制平仓，给持仓全天时间触达止盈。T+2 的持有期不应该只给 30 分钟的交易窗口。

### 4. 收紧尾盘拉升阈值

```
# 当前 (过低)
tail_up = cur_close > cur_open * 1.002

# 建议
tail_up = cur_close > cur_open * 1.005  # 至少 0.5%
```

同时可以考虑加入 OBV 确认作为替代方案：即使涨幅不够 0.5%，但如果 OBV 明显上升，也可以视为有效拉升。

### 5. Kronos 预测对齐持仓周期

将 Kronos 预测的 5 天平均改为取前 2 天预测的均值，与 T+2 持仓周期对齐：

```python
avg_pred = pred_df['close'].head(2).mean()  # 只取前 2 天
```

### 6. 涨幅打分分级

```
if 0.5 <= chg <= 1.5:   score += 20  # 温和上涨
elif 1.5 < chg <= 3.0:  score += 15  # 适中
elif 3.0 < chg <= 5.0:  score += 10  # 偏高（追高风险）
```

这与"尾盘选股不追高"的核心逻辑一致。
